> **Note**: This API provides comprehensive certification and badge management with verification, revocation, and audit capabilities.

## Overview

The Hixbe Certification & Badge API enables you to create, issue, verify, and manage digital certificates and badges. Perfect for educational institutions, certification bodies, and organizations needing to issue verifiable credentials.

## Authentication

All requests require Bearer token authentication:

```
Authorization: Bearer <sk_...>
```

## Base URLs

- Production: `https://api.hixbe.com/certify`

## Versioning

API versioning is handled via the `Hixbe-Version` header. Use `2026-01-01` for the latest version.

Include this header in all API requests:

```
Hixbe-Version: 2026-01-01
```

## Key Concepts

### Templates
Design certificate templates with custom backgrounds and field layouts.

### Credentials
Digital certificates and badges that can be issued to recipients.

### Badges
Digital badge definitions with metadata and criteria.

### Programs
Certification programs that group related credentials.

### Verification
Public verification endpoints for issued credentials.

### Revocation
Revoke credentials when they are no longer valid.

### Webhooks
Receive notifications about credential events.

### Audit Logs
Track all actions performed on credentials for compliance.

## Error Handling

All errors follow a consistent format with `error`, `message`, and optional `code` fields.

## Idempotency

Use the `Idempotency-Key` header for safe retries of POST requests.
